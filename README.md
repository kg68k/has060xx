# HAS060X.X

HAS060.X version 3.09+91の改造版です。  
無保証につき各自の責任で使用して下さい。


## 新機能

* HUPAIRに対応。
* 実行ファイル名の先頭を'g2as'(大文字小文字区別なし)にするとg2asモードで動作する。
  * 環境変数HASではなくG2ASを参照する。
  * `-1 -c4`オプションが自動指定される。
* 環境変数の先頭に'*'がなくても、常に'/'をファイル名の指定として扱う。
* プレデファインシンボル`__HAS060X__`(HAS060X.Xのバージョン)を追加。
* 1行の最大バイト数を8KBに増加。
* `MOVEM`命令のレジスタリストを即値形式でも記述できる。
* コマンドラインスイッチ`-_`を追加。オブジェクトファイルに出力される外部定義・外部
  参照シンボルの先頭に'_'を挿入する。
* コマンドラインスイッチ`-.`を追加。ドット(`.`)からはじまるラベルを許可する。
* コマンドラインスイッチ`-Werror`を追加。ワーニングをエラーとして扱う。
  (大文字小文字をこのとおりに記述しないと認識されない)
* `.extern`疑似命令。シンボル名を指定すると`.xref`と同様に外部参照宣言となるが、
  実際に参照されなかった場合はオブジェクトファイルには出力されない。
* `.include`、`.insert`疑似命令でエラーメッセージにファイル名を表示する。

### ドットラベル

* コマンドラインスイッチ`-.`指定時のみ使えるようになる。
* 定義する場合は行頭に空白を入れず記述し、直後に`:`を置く必要がある。
* ドットを除いた部分が命令、疑似命令、マクロ、演算子と同じものは使えない。
* 1文字のラベル(例: `.X`)も使用できるが、サイズ(`bwlsdxpq`)と同じものは使えない。
* ほかのアセンブラにあるようなローカルラベルとしての機能はない。
* 仕様を変更する可能性あり。


## 仕様の変更

* コマンドラインスイッチ`-f`の既定値を`-f0,0,96,58,16`に変更(改ページなし、幅96)。
* コマンドラインスイッチ`-j`の既定値を`-j3`に変更。
* コマンドラインスイッチ`-m`に対応MPU以外の数値を指定した場合はエラーにする。
* コマンドラインスイッチ`-d`でプレデファインシンボルを外部定義にしない。
* `.ds`疑似命令の長さに負数を指定した場合、.text/.dataセクション以外でもエラーとする。
* `.fequ`、`.fset`、`.reg`疑似命令でプレデファインシンボルを指定した場合のエラー表示を
  「プレデファインシンボル xxx を再定義しようとしました」に変更。
* `.fset`疑似命令以外で定義されたシンボルを`.fset`で再定義しようとした場合にエラーと
  する。コマンドラインスイッチ -j0 を指定するとワーニングの通知に格下げされる。
* `.dcb`疑似命令で長さが負数の場合のエラー表示を`.ds`と同じにした。
* `.dcb`疑似命令でアラインメントのワーニングよりエラーを優先して表示する。


## 削除された機能

* コマンドラインスイッチ`-8`、`-r`、`-z`。
* 別名命令`CLR An`、`DEC`、`INC`、`TPcc`。
* 疑似命令`68000`、`68010`、`68020`、`68030`、`68040`、`68060`。
* 別名レジスタ`r0`～`r15`、`zr0`～`zr15`。
* SXHAS機能。疑似命令`.rdata`、`.rbss`、`.rstack`、`.rcomm`、`.rldata`、`.rlbss`、
  `.rlstack`、`.rlcomm`。
* ColdFire対応。


## HAS060.X v3.09+91から修正された不具合

* `.comment`疑似命令でコメント終端が見つからないままファイルが終了すると
  存在しない行のアセンブルエラーが発生する。
* 同じ名称の命令や疑似命令がないマクロを定義し、マクロ名の先頭に`.`をつけて
  呼び出そうとすると「命令が解釈できません」のエラーになることがある。
* コマンドラインスイッチ`-y`と`-p`を同時に指定するとバスエラーが発生する。
* コマンドラインスイッチ`-call`の指定時に命令が展開されないことがある。
* シンボル定義時にワークメモリを破壊する(実害なし)。
* .cpu 68020のときエラッタF6の対策が有効にならない。
* `.defined.`演算子で検査したシンボルや`.reg`疑似命令でオペランドに記述したシンボル
  が未定義だと、未定義シンボルとして通知される。
* `.else`の後ろに`.else`や`.elseif`があってもエラーにならない。
  (ただし条件不成立ブロック内ではエラーチェックが省略される)
* `.if`などの条件不成立ブロック内で.commentが機能しない。
* `.dc.b 1.0`がエラーになる。
* `.insert`疑似命令でオフセットや長さの指定に間違いがあると、エラー後にファイルが
  開いたままになる。

## Build

PCやネット上での取り扱いを用意にするために、src/内のファイルはUTF-8で記述されています。
X68000上でビルドする際には、UTF-8からShift_JISへの変換が必要です。

### src2buildを使用する場合

必要ツール: [src2build](https://github.com/kg68k/src2build)

srcディレクトリのある場所で以下のコマンドを実行します。
```
src2build src
make -C build
```

### その他の方法

src/内のファイルを適当なツールで適宜Shift_JISに変換して別のディレクトリに保存し、
ディレクトリ内で`make`を実行してください。  
UTF-8のままでは正しくビルドできません。
